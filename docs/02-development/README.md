# 02 - Development Documentation

> **PASSIO Yazılım Geliştirme Standartları ve Mühendislik Kılavuzları**

---

## 🎯 Klasörün Amacı

Bu klasör; PASSIO projesinde kod kalitesini, sürdürülebilirliği ve mühendislik disiplinini sağlamak için kullanılan geliştirme standartlarını, kodlama kurallarını, Git iş akışlarını ve test stratejilerini barındırır.

---

## 📚 Burada Bulunacak Belgeler

- **Geliştirici İncili / Kılavuzu (`development-bible.md`):** Temel mühendislik prensipleri.
- **Kodlama Standartları (`coding-standards.md`):** TypeScript, React, Tailwind CSS yazım kuralları.
- **Git & PR İş Akışı (`git-workflow.md`):** Branch adlandırma, commit mesajı formatları ve PR onay süreçleri.
- **Hata Yönetimi & Logging (`error-handling.md`):** Domain hataları, log seviyeleri ve izlenebilirlik.
- **Test Stratejisi (`testing-strategy.md`):** Unit, Integration ve E2E test yaklaşımları.

---

## 🚀 Nasıl Kullanılacak?

1. **Yeni Katılan Geliştiriciler:** Projeye katkı sunmaya başlamadan önce buradaki tüm standartları okumalıdır.
2. **Kod Gözden Geçirme (PR Review):** PR kabul kriterleri doğrudan bu klasördeki kodlama ve Git kurallarına göre belirlenir.
3. **Refactoring & Bakım:** Kod iyileştirme süreçlerinde standartlardan sapma olup olmadığı kontrol edilir.

---

## ✍️ Belge Yazım Kuralları

- **Açıklık:** Kurallar somut kod örnekleri (`Do` / `Don't`) ile desteklenmelidir.
- **Format:** Standart Markdown (`.md`).
- **Dosya Adlandırma:** Kebab-case (Örn: `coding-standards.md`, `git-workflow.md`).
- **Güncellik:** Değişen kütüphane versiyonları veya geliştirme pratikleri doğrultusunda dokümanlar güncel tutulmalıdır.
